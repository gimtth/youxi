# 坐标问题修复说明

## 问题原因

截图区域与实际标注区域不一致，通常是由以下原因造成：
1. Windows DPI 缩放（125%, 150% 等）
2. 全屏截图后缩放显示导致的坐标转换误差
3. 多显示器环境下的坐标偏移

## 解决方案

现在提供两种方法获取准确的游戏区域坐标：

### 方法1：使用坐标获取工具（推荐）⭐

```bash
python get_region_coords.py
```

**步骤：**
1. 运行工具
2. 将鼠标移动到游戏区域左上角，按空格键
3. 将鼠标移动到游戏区域右下角，按空格键
4. 工具会显示坐标并保存验证截图 `verify_region.png`
5. 检查验证截图是否正确
6. 复制显示的坐标值

**输出示例：**
```
游戏区域坐标:
============================================================
左上角: (100, 200)
右下角: (600, 800)
区域尺寸: 500 x 600

复制以下内容用于 game_auto_player.py:
------------------------------------------------------------
x1 = 100
y1 = 200
x2 = 600
y2 = 800
------------------------------------------------------------
```

### 方法2：使用截图工具

1. 使用 QQ 截图、微信截图或其他工具
2. 截取游戏区域，工具会显示坐标和尺寸
3. 记录左上角和右下角的坐标

### 方法3：使用原有的可视化工具

```bash
python game_icon_matcher.py
```

运行后会显示坐标信息，但注意可能存在缩放误差。

## 使用坐标运行自动玩家

```bash
python game_auto_player.py
```

程序会提示输入坐标：
```
请输入左上角 X 坐标: 100
请输入左上角 Y 坐标: 200
请输入右下角 X 坐标: 600
请输入右下角 Y 坐标: 800
```

输入后会：
1. 显示区域尺寸
2. 截图并保存 `debug_screenshot.png`
3. 识别并保存 `debug_matched.png`
4. 检查这两个文件确认区域正确

## 验证坐标是否正确

运行后检查：
1. `debug_screenshot.png` - 应该包含完整的游戏区域
2. `debug_matched.png` - 应该标注了所有图标
3. 控制台输出的"截图尺寸"应该等于"预期尺寸"

如果不匹配：
- 检查是否有 DPI 缩放
- 尝试使用方法1重新获取坐标
- 确保游戏窗口没有移动

## 常见问题

**Q: 截图尺寸与预期不符？**
A: 可能是 DPI 缩放问题。使用 `get_region_coords.py` 工具，它会自动验证。

**Q: 识别到的图标数量不足？**
A: 
1. 检查 `debug_screenshot.png` 是否包含所有格子
2. 如果截图不完整，重新获取坐标
3. 如果截图完整但识别不全，降低阈值（已自动处理）

**Q: 多显示器环境？**
A: 使用 `get_region_coords.py` 工具，它使用绝对屏幕坐标，支持多显示器。

## 保存坐标配置（可选）

为了避免每次都输入坐标，可以修改 `game_auto_player.py`：

在 `initialize` 函数中直接设置：
```python
def initialize(self):
    # 直接使用保存的坐标
    x1, y1, x2, y2 = 100, 200, 600, 800  # 替换为你的坐标
    self.game_region = (x1, y1, x2, y2)
    
    print(f"使用保存的游戏区域: ({x1}, {y1}) 到 ({x2}, {y2})")
    return self.update_game_state(save_debug=True)
```
