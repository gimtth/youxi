核心思路
1. 游戏状态建模

将140个格子建模为一个网格（比如10x14或7x20）
每个格子存储：
- 图标类型（01-40）
- 位置坐标（相对于游戏区域）
- 中心点坐标（center_x, center_y）：用于精确点击和滑动
- 是否已消除（is_empty）

网格建立方法：
- 识别所有图标后，根据位置进行聚类分析
- 确定行数和列数
- 计算每个格子的中心点坐标
- 推算空格子的中心点位置（即使没有图标也能定位）
2. 寻找可消除的配对

分两种情况：

情况A：直接相邻可消除

扫描所有格子，检查上下左右四个方向
找到相同类型且相邻的配对
这是最优先的操作，无需移动
情况B：需要移动才能消除

对于每个图标类型，找出所有该类型的位置
如果同类型有2个或以上，计算它们之间的关系
判断通过滑动行或列能否让它们相邻
3. 移动策略优先级

从高到低：

零步消除：已经相邻的直接点击
一步消除：只需移动一次就能配对的
多步消除：需要多次移动的（可能引发连锁反应）
4. 移动可行性判断

需要考虑：

行滑动：检查该行是否可以整体或部分滑动
列滑动：检查该列是否可以整体或部分滑动
滑动距离：计算需要滑动多少格才能让两个相同图标相邻
障碍物：是否有其他因素阻止滑动
5. 高级策略

贪心策略：优先消除数量最多的图标类型（减少后期选择困难）
连锁反应：移动后可能让其他图标也变成相邻，一次操作多次消除
死局检测：如果没有任何可行操作，提示无解
回溯搜索：如果简单策略不行，用深度优先搜索找最优解
空格利用：消除后产生的空格可以作为滑动的"通道"，增加移动灵活性
6. 实现步骤

识别阶段：用现有脚本识别所有图标和位置

建立网格：根据位置信息构建游戏网格数据结构
- 对图标位置进行聚类，确定行列
- 计算每个格子的边界和中心点坐标
- 建立完整的坐标映射系统

寻找配对：遍历所有可能的消除机会

计算移动：对于每个配对，计算最少移动步数

执行操作：
- 如果是直接相邻：点击其中一个图标的中心位置
- 如果需要移动：从起始格子中心拖动到目标格子中心
- 所有鼠标操作都使用格子中心坐标，确保准确性

实时更新：每次操作后重新截图识别游戏区域，更新网格状态
原因：
- 消除后：对应位置变空
- 滑动后：多个砖块位置发生改变
- 必须重新识别才能获得准确的当前状态

循环直到通关或无解
7. 技术难点

网格对齐：需要准确判断每个图标属于哪一行哪一列
- 方法：对识别到的图标位置进行聚类（按Y坐标分行，按X坐标分列）
- 计算每行的平均高度和每列的平均宽度
- 建立完整的网格坐标系统

格子中心定位（重要）：
- 每个格子必须计算中心点坐标 (center_x, center_y)
- 点击操作：点击格子中心，避免误触边缘
- 滑动操作：从起始格子中心拖动到目标格子中心
- 空格子定位：即使格子为空，也能通过网格推算出中心位置
- 提高操作准确性和成功率

滑动检测：判断哪些砖块可以一起滑动（"部分相邻砖块"的规则需要实验）

移动模拟：
- 起点：起始格子的中心坐标
- 终点：目标格子的中心坐标
- 速度：适中的拖动速度，模拟人类操作

实时识别：每次操作后需要快速重新识别游戏区域
优化识别速度：
- 保存游戏区域坐标，避免重复选择
- 只截取游戏区域，不截全屏
- 可以只识别发生变化的行/列（高级优化）
8. 优化建议

先实现简单的"找相邻配对"功能
再逐步添加"一步移动"策略
最后考虑复杂的多步规划
可以加入人工干预：脚本提示最佳操作，由人来执行

9. 游戏特性（重要）

消除机制：
- 消除后对应位置变成空格，不会下落或填充
- 空格位置固定，直到游戏结束

滑动机制：
- 可以滑动整行或整列（或部分相邻砖块）
- 滑动会改变砖块位置：原位置变空，目标位置被填充
- 滑动可以利用空格作为"通道"

状态更新：
- 每次消除或滑动后，游戏状态都会改变
- 必须重新识别才能获得准确状态
- 优化方案：保存游戏区域坐标，只截取该区域进行识别


10. 坐标系统设计

数据结构示例：
```python
grid = {
    (row, col): {
        'type': '01',           # 图标类型，空格为None
        'center_x': 150,        # 格子中心X坐标（相对游戏区域左上角）
        'center_y': 200,        # 格子中心Y坐标（相对游戏区域左上角）
        'abs_center_x': 1150,   # 绝对屏幕坐标X（用于鼠标操作）
        'abs_center_y': 400,    # 绝对屏幕坐标Y（用于鼠标操作）
        'is_empty': False       # 是否为空格子
    }
}
```

坐标计算流程：
1. 识别所有图标，获得每个图标的位置 (x, y, width, height)
2. 计算每个图标的中心点：center = (x + width/2, y + height/2)
3. 按Y坐标聚类分组 → 确定行数和每行的Y坐标
4. 按X坐标聚类分组 → 确定列数和每列的X坐标
5. 为每个格子(row, col)计算中心点坐标
6. 空格子的中心点通过网格推算得出

鼠标操作使用绝对坐标：
- 点击：pyautogui.click(abs_center_x, abs_center_y)
- 滑动：pyautogui.drag(from_abs_x, from_abs_y, to_abs_x, to_abs_y)
